tida.scene.manager

Scene manager module.
Authors: TodNaz
License: MIT
class SceneManager;
Scene manager
@property @safe Scene[string] scenes();
List scenes
@property @safe Scene ofbegin();
The first added scene.
It can be overridden so that when the game is restarted, the manager will jump to the scene from this line: 
sceneManager.ofbegin = myScene;
sceneManager.gameRestart();
@property @safe Scene ofend();
The last added scene.
@property @safe Scene previous();
The previous scene that was active.
nothrow @property @safe Scene current();
Link to the currently active scene. With the help of it, it will be convenient for the instances to communicate with the scene. For more convenient communication, there is a function getScene, by which you can access the functions of the heir, but the main thing is to check the correctness of the scene, otherwise you can get an error: 
sceneManager.current.getScene!HeirScene.callback();
@property @safe Scene initable();
Temporary reference during initialization. Use it only in the init, restart, entry events. Contains a link to the current scene, which will be initialized (i.e., control is transferred).
@safe void trigger(string name);
Calls a trigger for the current scene, as well as its instances.
Parameters:
string name Trigger name.
@safe bool hasScene(Scene scene);
Checks if the scene is in the scene list.
Parameters:
Scene scene Scene.
@safe void add(Scene scene);
Adds a scene to the list.
Parameters:
Scene scene Scene.
@safe void add(T)();
Creates and adds a scene to the list.
Parameters:
T Scene name.
Example: 
sceneManager.add!MyScene;
@safe void close();
@safe void initThread(size_t count = 1);
@safe void pauseThread(size_t value);
@safe void inbegin();
Goes to the first scene added.
@safe void gotoin(string name);
Goes to the scene by its string name.
Parameters:
string name Scene name.
@safe void gotoin(Scene scene);
Goes to the scene by his heir name.
Parameters:
Scene scene Scene heir.
